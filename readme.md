# 📸 Instagram 混合模式下载器使用手册

本工具采用 **“Selenium 提取链接 + Instaloader 静态下载”** 的混合模式，专门解决直接运行爬虫容易触发 `429 Too Many Requests` 的问题。

---

## 🛠 1. 环境准备

在开始之前，请确保你的电脑已安装 **Python 3.8+** 以及 **Chrome 浏览器**。

### 安装依赖库

打开终端（Terminal/Command Prompt），运行以下命令：

```bash
pip install selenium instaloader requests webdriver-manager

```

---

## 🚀 2. 操作流程

本工具由两个核心脚本组成，必须按顺序执行：

### 第一步：获取权限 (运行 `auth.py`)

这是为了让程序拥有你的登录状态，避免下载高清图时被拦截。

1. 运行脚本：`python auth.py`
2. 在弹出的 Chrome 浏览器中手动输入账号、密码登录。
3. 登录成功进入 Instagram 首页后，回到命令行按 **回车**。
4. 文件夹下会生成 `cookies.pkl` 文件。

### 第二步：精准下载 (运行 `unified_downloader.py`)

1. 运行脚本：`python unified_downloader.py`
2. **输入目标**：输入你想爬取的账号 ID（例如：`jadeuly713`）。
3. **选择模式**：
* **模式 1**：下载最近的 $N$ 条帖子。
* **模式 2**：下载指定范围（例如第 3 到第 10 条）。
* **模式 3**：直接粘贴单条帖子的链接下载。



---

## 📂 3. 运行原理说明

1. **Selenium 模块**：模拟真人打开浏览器并滚动页面。由于 Instagram 对“浏览器访问”的审查比“API访问”宽松，这步能安全地拿到详情页的 `shortcode`（帖子指纹）。
2. **Instaloader 模块**：获取链接后，程序关闭浏览器，切换到轻量级的下载引擎。通过注入第一步保存的 `Cookies`，以合法身份高速获取原图和视频。

## ⚠️ 4. 反爬避坑指南（非常重要）

为了保护你的账号不被封禁/限制，请务必遵守以下原则：

| 风险项 | 建议 | 原因 |
| --- | --- | --- |
| **下载总量** | 单次建议不超过 **30** 篇 | 大批量请求会触发 IG 针对账号的行为异常检测。 |
| **执行频率** | 两次任务间隔 **10** 分钟以上 | 连续频繁运行脚本会导致 IP 被暂时标记。 |
| **大休息机制** | 每 5 篇休息 **60** 秒 | 脚本已内置此逻辑，请勿人为删除 `time.sleep`。 |
| **出现 429 报错** | **立即停止**，等待 2-24 小时 | 这是 IG 最严重的警告，强行尝试会导致封号。 |

## ❓ 5. 常见问题 (FAQ)

**Q: 为什么我运行后浏览器一闪而过？**
A: 请检查是否已关闭之前运行的 Chrome 进程，或尝试更新 Chrome 浏览器到最新版本。

**Q: 下载的内容保存在哪里？**
A: 默认会保存在项目根目录下的 `downloads/账号名/` 文件夹中。

**Q: 能够下载私密账号吗？**
A: 只要你在第一步 `auth.py` 中登录的账号已经关注了该私密账号，就可以正常下载。

## 📝 6. 开发者贴士

* 如果需要下载 **视频**，请确保你的网络环境可以正常播放 IG 视频。
* 建议定期重新运行 `auth.py`，因为 Instagram 的登录有效期（Cookies）通常只有 7-14 天。
